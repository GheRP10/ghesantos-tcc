6. REQUISITOS DO SISTEMA

6.1 Requisitos Funcionais

RF01 — Cadastro de Clientes

O sistema permite o cadastro de clientes com histórico de atendimentos e mecanismos de busca, oferecendo flexibilidade conforme a necessidade do atendimento e preferência do cliente.

Modalidades de Cadastro:

Cadastro Simples:

  • Dados obrigatórios: Nome completo e telefone principal;
  • Dados opcionais: Contato de referência;
  • Finalidade: Atendimento rápido para clientes que não necessitam de documentação fiscal;
  • Documentação: Recibo de serviço para garantia;
  • Uso: Ideal para reparos simples e clientes que preferem agilidade.

Cadastro Completo:

  • Dados obrigatórios: Nome completo e telefone principal, CPF, endereço completo, e-mail;
  • Dados opcionais: Contato de referência;
  • Finalidade: Clientes que necessitam de documentação completa ou nota fiscal;
  • Documentação: Permite futuras integrações com sistemas de emissão fiscal;
  • Uso: Ideal para clientes corporativos ou que demandam faturamento.

Esta flexibilidade reflete o foco do sistema GS CELL na gestão operacional das ordens de serviço, mantendo a separação de responsabilidades entre controle operacional e emissão fiscal, conforme boas práticas de arquitetura de software.

RF02 — Cadastro de Ordens de Serviço

O sistema permite o registro detalhado de ordens de serviço incluindo informações do cliente, descrição do dispositivo, problema relatado, orçamento preliminar e prazo de entrega estimado.

Controle de Status:
  • OSO (Ordem de Serviço Orçamento): Ordem gerada para cálculo de custo;
  • OSS (Ordem de Serviço Solicitada): Cliente aprova o serviço e solicita o conserto;
  • OSA (Ordem de Serviço em Andamento): Técnicos estão executando o reparo;
  • OSF (Ordem de Serviço Finalizada): Conserto concluído e aguardando retirada pelo cliente;
  • OSE (Ordem de Serviço Entregue): Aparelho reparado e entregue ao cliente;
  • OSC (Ordem de Serviço Cancelada): Serviço cancelado antes da execução.
RF03 — Cadastro de Funcionários

O sistema permite o cadastro, edição e inativação de funcionários que atuam no processo operacional. Cada funcionário deve conter dados de identificação (nome, sobrenome, CPF, telefone, cargo, e-mail) e é vinculado às operações realizadas no sistema. O cargo do funcionário definirá seu nível de acesso às funcionalidades (ex.: administrador, gerente, técnico, atendente).

RF04 — Cadastro de Usuários

O sistema permite o cadastro, edição e inativação de usuários responsáveis por acessar o sistema.

Cada usuário estará obrigatoriamente vinculado a um funcionário previamente cadastrado.

O login de acesso é gerado automaticamente combinando o nome, o sobrenome e os três primeiros dígitos do CPF do funcionário vinculado.

O sistema permite a definição e alteração de senha, com armazenamento seguro utilizando técnicas de criptografia.

O nível de acesso do usuário é definido com base no cargo do funcionário vinculado.

RF05 — Cadastro de Produtos

O sistema permite o registro e controle de estoque de produtos e componentes utilizados nos reparos, incluindo preço de custo, preço de venda e quantidade disponível.

Controle de Estrutura:

Grupo Produto: Utilizados para categorizar os produtos por tipo:

  • Telas; Baterias; Acessórios.

Unidade: Define a forma de quantificação dos produtos:

  • UN (Unidade); CX (Caixa); PC (Peça).
RF06 — Vinculação de Produtos à Ordem de Serviço

O sistema permite a vinculação de produtos às ordens de serviço, registrando os itens utilizados em cada atendimento e realizando a baixa automática no estoque.

RF07 — Controle Financeiro

O sistema permite registrar valores orçados, adiantamentos recebidos, pagamentos finais e formas de pagamento, gerando um controle financeiro básico.

RF08 — Cadastro de Aparelhos

O sistema permite o cadastro e gerenciamento de aparelhos, incluindo marca, modelo, tipo e observações, para facilitar o vínculo às ordens de serviço.

RF09 — Registro de Operações (Log de Sistema)

O sistema permite manter um histórico das operações realizadas pelos usuários, incluindo data, hora, tipo de ação e identificação do usuário responsável.

6.2 Requisitos Não Funcionais

RNF01 — Usabilidade

O sistema deve apresentar interface intuitiva e responsiva, adaptável a diferentes dispositivos e tamanhos de tela, com tempo de aprendizado reduzido.

RNF02 — Desempenho

O sistema deve responder às requisições em tempo máximo de 3 segundos em condições normais de utilização no ambiente local.

RNF03 — Disponibilidade

O sistema deve estar disponível durante o período de operação do ambiente local, considerando manutenções programadas.

RNF04 — Segurança

O sistema deve implementar autenticação segura, controle de acesso baseado em perfis, criptografia de dados sensíveis e proteção contra ataques comuns (SQL Injection, XSS).

RNF05 — Escalabilidade

O sistema deve ser capaz de suportar o crescimento da base de dados e do número de usuários sem degradação significativa de desempenho.

RNF06 — Portabilidade

O sistema deve funcionar corretamente nos principais navegadores web (Chrome, Firefox, Safari, Edge) em suas versões mais recentes.

RNF07 — Manutenibilidade

O código deve ser modular, bem documentado e seguir padrões de desenvolvimento que facilitem sua manutenção e evolução.

RNF08 — Compatibilidade

O sistema foi projetado de forma a permitir compatibilidade com impressoras térmicas padrão de mercado.

RNF09 — Formato de Relatórios

Os relatórios gerados pelo sistema devem ser disponibilizados em formato PDF, garantindo padronização e portabilidade das informações.

RNF10 — Rastreabilidade

O sistema deve garantir a rastreabilidade das ações realizadas, permitindo identificar os funcionários responsáveis por cada etapa do processo.

RNF11 — Histórico de Alterações

O sistema mantém um log de todas as alterações realizadas nas ordens de serviço, identificando usuário, data, hora e modificação realizada.

RNF12 — Busca e Filtros

O sistema deve oferecer mecanismos avançados de busca e filtros para localização rápida de ordens de serviço com base em diversos parâmetros.
